2. Log Inventory

Score each log source you collect (0 = none, 5 = excellent). Log sources roll up to data components and on through analytics → detection strategies → techniques. Saved to your browser automatically.

Samples: mature · cloud · mssp · greenfield
+ Add log source by hand (event ID / channel / SIEM-specific)

Useful when your tool emits a log under a different name than the bundle's defaults — e.g. winlogbeat / Security/4688, edr-vendor / process_creation, or a Sigma-style product/category/service. If the (name, channel) matches a log source the bundle already knows about, your score lands on that record and immediately drives coverage. Otherwise it's stored as a custom entry so you can keep track of it.

3. Data Components

Flat list of every ATT&CK data component. The score on the right is what you set on the Log Inventory tab. Use this to spot collection gaps quickly.

4. Detection strategies

Detection strategies (top) bundle one or more analytics, each tied to specific log sources. Lit strategies surface the techniques you can detect (table below).

5. Threats

Pick the MITRE ATT&CK threat-actor groups (e.g. APT29, FIN7) you care about. The next tab compares their techniques against your detection coverage.

Samples: mixed · ransomware crews · state APTs · financial

6. Coverage

Cross-references the techniques used by the threat groups you picked against your detection strategies. Gaps are techniques those groups use that your data sources can't catch.
